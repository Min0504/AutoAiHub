n8n 무료로 쓰는 법
셀프호스팅 완전 정복 (2026)

n8n 클라우드는 월 $20부터 시작합니다. 하지만 셀프호스팅하면 서버비만 내고 사용량 제한 없이 쓸 수 있습니다. 무료 서버 옵션도 있고요.

AutoHub AI 2026년 6월 읽기 약 8분

n8n은 오픈소스 자동화 툴입니다. 소스코드가 공개되어 있고, 직접 서버에 설치해서 쓸 수 있습니다. 설치형을 쓰면 사용량 제한이 없고, 데이터가 외부로 나가지 않으며, 비용은 서버비가 전부입니다.

이 글에서는 세 가지 방법으로 n8n을 무료 또는 저렴하게 설치하는 방법을 설명합니다.

로컬 Docker무료하개인 테스트, 로컬 업무

방법 1 — 로컬 Docker (가장 쉬움)

내 컴퓨터에 설치해서 쓰는 방법입니다. 컴퓨터가 켜져 있는 동안만 동작하지만, 테스트나 개인 자동화에는 충분합니다.

사전 조건: Docker Desktop 설치 (docker.com에서 무료 다운로드)

# n8n 실행 (데이터 영구 보존 포함)
docker run -it --rm \
  --name n8n \
  -p 5678:5678 \
  -v ~/.n8n:/home/node/.n8n \
  docker.n8n.io/n8nio/n8n

실행 후 브라우저에서 http://localhost:5678을 열면 n8n 에디터가 나옵니다. 처음 접속 시 이메일/비밀번호로 계정을 만들면 됩니다.

💡 컴퓨터를 재시작해도 데이터가 유지되도록 -v ~/.n8n:/home/node/.n8n 옵션이 들어가 있습니다.

방법 2 — Railway 배포 (가장 간단한 클라우드)

Railway는 GitHub 계정으로 로그인해서 클릭 몇 번으로 서버에 배포할 수 있는 플랫폼입니다. 월 $5 크레딧을 무료 제공하는데, n8n 혼자 쓰기엔 충분합니다.

1

Railway 계정 만들기

railway.app에서 GitHub 계정으로 로그인합니다.

2

n8n 템플릿 배포

Railway 마켓플레이스에서 "n8n"을 검색하거나, 아래 Docker Image로 직접 배포합니다.

Image: docker.n8n.io/n8nio/n8n

3

환경 변수 설정

N8N_BASIC_AUTH_ACTIVE=true
N8N_BASIC_AUTH_USER=admin
N8N_BASIC_AUTH_PASSWORD=비밀번호입력
WEBHOOK_URL=https://your-app.railway.app
4

배포 후 접속

Railway가 자동으로 HTTPS URL을 생성합니다. 해당 URL로 접속하면 n8n 에디터가 열립니다.

방법 3 — Oracle Cloud Free Tier (가장 강력한 무료)

Oracle Cloud는 AMD 1코어 + 1GB RAM VM을 영구 무료로 제공합니다. n8n이 항상 켜져 있어야 한다면 이 방법이 최선입니다.

1

Oracle Cloud 계정 생성

oracle.com/cloud/free에서 가입합니다. 신용카드 인증이 필요하지만 무료 티어에서는 절대 과금이 되지 않습니다.

2

VM 생성

Compute → Instances → Create Instance. Shape: VM.Standard.E2.1.Micro (Always Free 표시). OS: Ubuntu 22.04.

3

Docker + n8n 설치

# SSH 접속 후
sudo apt update && sudo apt install -y docker.io
sudo systemctl start docker
sudo systemctl enable docker

# n8n 실행
sudo docker run -d \
  --name n8n \
  --restart always \
  -p 5678:5678 \
  -v ~/.n8n:/home/node/.n8n \
  -e N8N_BASIC_AUTH_ACTIVE=true \
  -e N8N_BASIC_AUTH_USER=admin \
  -e N8N_BASIC_AUTH_PASSWORD=비밀번호 \
  docker.n8n.io/n8nio/n8n
4

방화벽 포트 열기

Oracle Cloud 콘솔 → VCN → Security List에서 포트 5678 인바운드를 허용합니다. 이후 VM의 공인 IP로 접속 가능합니다.

⚠️ 외부에서 접근 가능하게 열 때는 반드시 기본 인증(Basic Auth)을 켜거나 Nginx로 HTTPS + 인증을 추가하세요.

설치 후 — 첫 워크플로우 만들기

n8n 에디터에 처음 들어가면 빈 캔버스가 나옵니다. 워크플로우를 만드는 기본 구조는 n8n 클라우드와 완전히 동일합니다.

  • 트리거 노드: 언제 실행할지 결정 (웹훅, 스케줄, 앱 이벤트 등)
  • 액션 노드: 무엇을 할지 결정 (HTTP 요청, DB 쓰기, 메시지 전송 등)
  • 코드 노드: JavaScript로 데이터 가공 (Make의 함수 대비 훨씬 유연)

400개 이상의 내장 앱 연동이 있고, HTTP Request 노드로 API가 있는 서비스는 뭐든 연결할 수 있습니다. AI 에이전트 기능도 내장되어 있어서 n8n을 LLM 기반 자동화에도 쓸 수 있습니다.

셀프호스팅의 가장 큰 장점은 비용만이 아닙니다. 내부 시스템이나 VPN 뒤의 서비스와 직접 연결할 수 있고, 고객 데이터가 외부 서버를 거치지 않습니다. 보안 요구사항이 있는 팀에게 사실상 필수입니다.
