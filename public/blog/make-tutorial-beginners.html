첫 자동화 만들기 (2026)

Make는 코딩 없이 앱과 서비스를 연결해 반복 업무를 자동화하는 툴입니다. 계정 만들기부터 실제로 동작하는 첫 자동화를 완성하기까지 단계별로 설명합니다.

AutoHub AI 2026년 6월 읽기 약 10분

Make(구 Integromat)는 현재 노코드 자동화 툴 중 가장 균형 잡힌 선택지입니다. 시각적인 버블 UI 덕분에 워크플로우 흐름이 한눈에 보이고, 무료 플랜이 넉넉해서 부담 없이 시작할 수 있습니다.

이 가이드에서는 "구글 폼 제출 → 슬랙 알림" 이라는 가장 간단한 예제를 만들면서 Make의 핵심 개념을 배웁니다.

Make 핵심 개념 3가지

Make를 처음 열면 낯선 UI에 당황할 수 있습니다. 하지만 알아야 할 개념은 세 가지뿐입니다.

  • 시나리오 (Scenario): 자동화 하나. "A가 발생하면 B를 한다"는 규칙 하나가 시나리오입니다.
  • 모듈 (Module): 시나리오를 구성하는 블록. 각 앱(구글, 슬랙, 노션 등)에서 제공하는 동작 단위입니다.
  • 오퍼레이션 (Operation): 모듈이 실행될 때마다 1 오퍼레이션 소비. 무료 플랜은 월 1,000 오퍼레이션입니다.
시나리오는 조리법, 모듈은 재료, 오퍼레이션은 요리할 때 쓰는 재료 한 개라고 생각하면 됩니다.

1단계 — Make 계정 만들기

1

가입

make.com에서 이메일 또는 구글 계정으로 가입합니다. 신용카드 없이 무료로 시작됩니다.

2

Organization 이름 입력

가입 후 "조직 이름"을 묻는 화면이 나옵니다. 개인이면 본인 이름이나 아무 텍스트나 입력해도 됩니다.

3

대시보드 도착

왼쪽 사이드바에 Scenarios, Connections, Data Stores 등이 보입니다. 핵심은 Scenarios입니다.

2단계 — 첫 시나리오 만들기

예제: 구글 폼에 응답이 오면 슬랙 채널에 알림을 보냅니다.

1

새 시나리오 생성

대시보드 우측 상단 "Create a new scenario" 클릭. 흰 캔버스에 물음표가 있는 동그란 원이 나타납니다. 이게 첫 번째 모듈(트리거)입니다.

2

트리거 모듈 설정 — Google Forms

원을 클릭하면 앱 검색창이 열립니다. "Google Forms"를 검색하고 클릭합니다.

트리거 목록에서 "Watch Responses"를 선택합니다. 구글 계정을 연결(Connect)하고, 감시할 폼을 선택합니다.

"Where to start" 질문에는 "From now on"을 선택하세요. 설정 후 OK.

3

액션 모듈 추가 — Slack

트리거 모듈 오른쪽에 + 버튼을 클릭합니다. "Slack"을 검색하고 "Create a Message"를 선택합니다.

슬랙 워크스페이스를 연결하고, 알림을 보낼 채널을 선택합니다.

메시지 내용에 구글 폼의 응답 데이터를 넣고 싶다면, 텍스트 입력창에서 왼쪽 동그란 아이콘을 클릭하면 폼 응답 데이터(이름, 이메일 등)를 선택해서 삽입할 수 있습니다. OK.

4

테스트 실행

왼쪽 하단 "Run once" 버튼을 클릭합니다. Make가 "지금 실행 중이니 구글 폼에 응답을 제출해보세요"라는 상태가 됩니다.

구글 폼에 실제로 응답을 제출합니다. 잠시 후 슬랙에 메시지가 오면 성공입니다.

5

스케줄 활성화

테스트 성공 후 왼쪽 하단 토글 스위치(Scheduling)를 ON으로 켭니다. 이제 구글 폼에 응답이 올 때마다 자동으로 슬랙 알림이 갑니다.

💡 Make는 기본적으로 15분 간격으로 새 트리거를 확인합니다. 더 빠르게 하려면 유료 플랜에서 간격을 줄일 수 있습니다. 즉시 실행이 필요하면 웹훅(Webhook) 트리거를 쓰세요.

3단계 — 조건 추가하기 (Filter)

모든 응답이 아닌, 특정 조건을 만족하는 경우만 슬랙에 보내고 싶다면 필터를 추가합니다.

두 모듈 사이의 연결선에 마우스를 올리면 렌치 아이콘이 나타납니다. 클릭하면 필터 설정창이 열립니다. 예를 들어 "이름이 '긴급'을 포함할 때만" 같은 조건을 추가할 수 있습니다.

Make를 200% 활용하는 팁

데이터 변환 — 함수 사용

Make에는 내장 함수가 있습니다. 날짜 형식 변환(formatDate), 텍스트 자르기(substring), 숫자 계산 등을 코딩 없이 할 수 있습니다. 모듈 설정창에서 데이터를 입력할 때 함수 버튼을 클릭하면 목록이 나옵니다.

오류 핸들링 — Error Handler

모듈에서 오류가 발생했을 때 어떻게 처리할지 정할 수 있습니다. 모듈 우클릭 → "Add error handler"로 추가합니다. 실패 시 이메일 알림을 보내거나, 다른 경로로 분기할 수 있습니다.

실행 이력 보기

시나리오 상세 화면에서 History 탭을 클릭하면 지금까지 실행된 기록이 모두 나옵니다. 어떤 데이터가 들어와서 어떻게 처리됐는지 단계별로 확인할 수 있습니다. 오류 발생 시 정확히 어떤 모듈에서 실패했는지 바로 볼 수 있습니다.

무료 플랜 한도를 아끼는 방법

  • 스케줄 간격을 늘린다: 15분 → 1시간으로 바꾸면 오퍼레이션 소비가 4분의 1로 줄어듭니다.
  • Router로 한 번에 분기: 동일한 트리거에서 여러 액션을 할 때, Router 모듈로 분기하면 트리거 오퍼레이션을 한 번만 씁니다.
  • Aggregator로 묶기: 여러 데이터를 하나로 합쳐서 한 번에 처리하면 오퍼레이션 절약이 됩니다.

⚠️ 월 1,000 오퍼레이션이 부족하면 Core 플랜($9/월, 10,000 오퍼레이션)으로 업그레이드가 현실적입니다. 한 달 커피 한 잔 값에 웬만한 자동화는 다 커버됩니다.

Make는 처음 30분이 고비입니다. 첫 시나리오가 동작하는 걸 보면 "이것도 자동화할 수 있겠다"는 아이디어가 계속 나오기 시작합니다. 지금 바로 Make 무료 계정을 만들고 위 예제를 따라해보세요.
